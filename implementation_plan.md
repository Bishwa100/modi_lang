# 🇮🇳 Modi Lang — The Deshbhakt Programming Language

> _"Mitron, aaj se coding bhi deshbhakti hai!"_

A hilarious, fully functional programming language where every keyword is a famous Modi catchphrase. Built as a tree-walking interpreter in **Node.js**.

---

## 🎯 Concept

Modi Lang is a **satirical esoteric programming language** that replaces boring keywords like `if`, `else`, `print`, and `function` with iconic Modi-isms. It's funny, it's functional, and it compiles your patriotism into actual programs.

---

## 🗣️ Language Keywords & Syntax

### Core Keywords Mapping

| Normal Keyword | Modi Lang Keyword | Why It's Funny |
|---|---|---|
| `let` / `var` | `modi` | _"Modi hai toh variable mumkin hai"_ — Modi creates everything |
| `print` | `mann_ki_baat` | _He speaks to the nation_ — now your program does too |
| `if` | `agar_modi` | _"Agar Modi chahein toh..."_ — conditional on Modi's will |
| `else` | `nahi_toh` | _"Nahi toh kya hoga?"_ — the alternative |
| `else if` | `warna_modi` | _Another condition, another promise_ |
| `true` | `acche_din` | _Good days = True. Always promised, sometimes delivered_ |
| `false` | `jumla` | _Jumla = False. Self-explanatory_ 😂 |
| `while` | `ghuma_ke` | _"Ghuma ke maarta hai"_ — loops around and around |
| `for` | `sabka_saath` | _"Sabka saath, sabka vikas"_ — iterate with everyone |
| `function` | `mitron` | _Every speech starts with "Mitron..."_ — every function too |
| `return` | `laut_aao` | _"Ghar wapsi"_ — return home with a value |
| `null` / `none` | `nota` | _NOTA = None of the Above_ — perfect for null |
| `input` | `chai_pe_charcha` | _"Chai pe charcha"_ — take input over chai_ |
| `import` | `make_in_india` | _Import? No! Make in India!_ |
| `try` | `modi_hai_toh` | _"Modi hai toh mumkin hai"_ — try it!_ |
| `catch` | `pakoda` | _When things go wrong, sell pakodas_ 🍳 |
| `break` | `demonetization` | _Breaks the loop just like it broke the economy_ 💸 |
| `continue` | `aage_badho` | _"Aage badho"_ — skip and move forward |
| `class` | `vishwaguru` | _World leader_ — the supreme class |
| `new` | `naya_bharat` | _"Naya Bharat"_ — create new instances |
| `delete` / `clear` | `swachh_bharat` | _Clean up!_ 🧹 |
| `end program` | `jai_hind` | _Every program ends with patriotism_ |
| `start program` | `namaste` | _Begin with Indian greeting_ 🙏 |
| `else` (block) | `bas_kar` | _"Bas kar!"_ — stop execution immediately |
| `length` | `lambai` | _"Length in Hindi"_ — measure arrays/strings |
| `push` | `jodo_rally` | _"Add to rally"_ — append to array |
| `pop` | `nikalo` | _"Remove"_ — pop from array |
| `type check` | `type_kya_hai` | _"What type is this?"_ — check variable type |
| `AND` | `aur` | _And in Hindi_ |
| `OR` | `ya` | _Or in Hindi_ |
| `NOT` | `bilkul_nahi` | _"Absolutely not!"_ |
| `==` | `same2same` | _Equality check, desi style_ |
| `!=` | `jumla_hai` | _"That's a jumla!"_ — not equal |

| `type conversion` | `aankda()` / `bhaashan()` | _Convert to number/string_ |
| `string split` | `split()` | _Split string by delimiter_ |
| `uppercase` | `uppercase()` | _Convert to uppercase_ |
| `lowercase` | `lowercase()` | _Convert to lowercase_ |
| `absolute` | `abs()` | _Absolute value_ |
| `power` | `power(a, b)` | _a to the power of b_ |

### Data Types

| Type | Modi Lang Name | Example |
|---|---|---|
| Number | `Aankda` | `modi votes = 303;` |
| String | `Bhaashan` (Speech) | `modi msg = "Mitron!";` |
| Boolean | `Acche_din / Jumla` | `modi sach = acche_din;` |
| Array | `Rally` | `modi crowd = [1, 2, 3];` |
| Null | `Nota` | `modi result = nota;` |
| Function | `Mitron` | `mitron myfunc() { ... }` |

### Operators

| Operation | Modi Lang | Example |
|---|---|---|
| Addition | `jodo` or `+` | `a + b` or `a jodo b` |
| Subtraction | `ghata` or `-` | `a - b` or `a ghata b` |
| Multiply | `guna` or `*` | `a * b` |
| Divide | `baanto` or `/` | `a / b` |
| Modulo | `baaki` or `%` | `a % b` |
| Assignment | `=` | `modi x = 10;` |

## 🏗️ Architecture — Interpreter Design

```
┌──────────────┐     ┌──────────┐     ┌──────────┐     ┌───────────┐
│  Source Code  │────▶│  Lexer   │────▶│  Parser  │────▶│ Evaluator │
│  (.modi)     │     │ (Tokens) │     │  (AST)   │     │ (Execute) │
└──────────────┘     └──────────┘     └──────────┘     └───────────┘
                                                             │
                                                             ▼
                                                       ┌───────────┐
                                                       │  Output   │
                                                       │ (Console) │
                                                       └───────────┘
```

### Components

1. **Lexer (Tokenizer)** — Reads `.modi` source files and converts them into tokens
   - Maps Modi keywords → token types
   - Handles strings, numbers, identifiers, operators
   - Tracks line/column for hilarious error messages

2. **Parser** — Converts token stream into an AST (Abstract Syntax Tree)
   - Handles expressions, statements, blocks
   - Validates `namaste` ... `jai_hind` program structure
   - Supports operator precedence

3. **Evaluator (Interpreter)** — Walks the AST and executes the program
   - Variable scoping (function-level)
   - Built-in functions (`mann_ki_baat`, `chai_pe_charcha`)
   - Funny runtime error handling

4. **REPL** — Interactive mode for quick experiments
   - Colorful terminal UI with Modi-themed prompt
   - `>>> Mitron, kya bolte ho? >`

## 🚀 Phased Build Plan

### Phase 1: Foundation (Lexer + Tokens) 
- Define all token types and keyword mappings
- Build the lexer to tokenize `.modi` files
- Handle strings, numbers, identifiers, operators
- Implement line/column tracking for error messages
